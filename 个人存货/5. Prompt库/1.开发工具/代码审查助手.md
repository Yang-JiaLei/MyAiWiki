# 代码审查助手

用于代码审查的专业提示词，帮助发现代码质量问题、潜在bug和安全漏洞。

## 提示词内容

```
请作为代码审查助手，帮我审查以下代码。请关注：

1) **代码质量**
   - 代码可读性和可维护性
   - 代码结构是否合理
   - 是否有重复代码
   - 命名规范是否一致

2) **潜在bug**
   - 边界条件处理
   - 空指针/未定义值处理
   - 资源泄漏风险
   - 并发安全问题

3) **性能问题**
   - 算法复杂度
   - 内存使用效率
   - I/O操作优化
   - 数据库查询优化

4) **安全漏洞**
   - 输入验证和过滤
   - SQL注入风险
   - XSS攻击风险
   - 敏感信息泄露

5) **代码风格**
   - 是否符合项目规范
   - 注释是否充分
   - 错误处理是否完善
   - 测试覆盖是否足够

请提供具体的改进建议，包括：
- 问题描述
- 风险等级（高/中/低）
- 具体位置（行号）
- 改进方案
- 示例代码（如果需要）

审查的代码：
[在此处粘贴需要审查的代码]
```

## 使用示例

### 输入
```python
def process_data(data):
    result = []
    for item in data:
        if item > 0:
            result.append(item * 2)
    return result
```

### 输出
```
代码审查报告：

1) 代码质量 - 良好
   - 代码简洁清晰
   - 函数单一职责

2) 潜在bug - 低风险
   - 缺少输入验证（data可能为None）
   - 建议添加类型提示

3) 性能问题 - 无
   - 算法复杂度O(n)，合理

4) 安全漏洞 - 无
   - 无外部输入，安全

5) 代码风格 - 良好
   - 符合Python PEP8规范

改进建议：
1. 添加输入验证和类型提示
   ```python
   from typing import List, Optional
   
   def process_data(data: Optional[List[float]]) -> List[float]:
       if data is None:
           return []
       result = []
       for item in data:
           if item > 0:
               result.append(item * 2)
       return result
   ```

2. 考虑使用列表推导式
   ```python
   def process_data(data: Optional[List[float]]) -> List[float]:
       if data is None:
           return []
       return [item * 2 for item in data if item > 0]
   ```
```

4. **学习提高**：通过审查学习最佳实践

## 效果评估

### 优点
1. **全面性**：覆盖代码质量的多个维度
2. **具体性**：提供具体的改进建议
3. **实用性**：可直接应用于实际开发
4. **教育性**：帮助开发者提高编码水平

### 改进空间
1. **语言特定**：可能需要针对不同编程语言调整
2. **上下文依赖**：需要了解项目背景和规范
3. **复杂度限制**：对于非常复杂的代码可能需要分块审查
