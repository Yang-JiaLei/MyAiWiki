# 个人Prompt库

基于 prompts.chat 模式构建的个人提示词库，收集和整理我在日常工作和学习中使用的各种AI提示词。

## 概述

这是一个结构化的提示词管理系统，旨在：
- 收集和整理高质量的AI提示词
- 按照使用场景和功能分类
- 支持快速搜索和复用
- 记录提示词的使用效果和改进

## 目录结构

```
个人Prompt库/
├── 1. 开发工具/          # 编程、调试、代码相关提示词
├── 2. 写作助手/          # 文案、创作、翻译提示词
├── 3. 学习研究/          # 学习、研究、分析提示词
├── 4. 工作效率/          # 办公、管理、效率提示词
├── 5. 创意娱乐/          # 创意、娱乐、生活提示词
├── 6. 专业领域/          # 专业领域特定提示词
├── 7. 系统指令/          # 系统级、配置指令
├── 8. 模板库/            # 可复用的提示词模板
├── 9. 效果评估/          # 提示词使用效果记录
├── prompts.csv           # 所有提示词的CSV数据库
└── README.md            # 本文件
```

## 使用指南

### 1. 添加新提示词

使用标准格式添加新提示词：

```markdown
## 提示词名称
**场景**: [使用场景]
**模型**: [适用的AI模型]
**标签**: [标签1, 标签2, ...]
**效果**: [使用效果评分 1-5]

### 提示词内容
[完整的提示词内容]

### 使用示例
[使用示例和输出]

### 改进建议
[可能的改进方向]
```

### 2. 搜索提示词

- 按场景搜索：查看对应目录
- 按标签搜索：使用Obsidian标签功能
- 按效果搜索：查看效果评估记录

### 3. 评估和改进

每个提示词都应记录：
- 使用次数
- 成功率
- 改进历史
- 用户反馈

## 数据格式

### CSV格式 (prompts.csv)
```
act,prompt,for_devs,type,contributor,created_at,updated_at,tags,effectiveness
[角色],[完整提示词],[是否开发者],[类型],[创建者],[创建时间],[更新时间],[标签],[效果评分]
```

### Markdown格式
每个提示词单独一个Markdown文件，包含完整的元数据和内容。

## 集成工具

### 1. Obsidian集成
- 使用Dataview插件查询提示词
- 使用Templates插件快速创建
- 使用Tag插件进行分类管理

### 2. 外部工具集成
- 可导出为JSON/CSV格式
- 支持API访问
- 可集成到AI助手工具链
