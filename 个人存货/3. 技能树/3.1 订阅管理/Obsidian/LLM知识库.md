## 内容摘要

这篇文章围绕 Karpathy 的 `LLM Wiki` 思路展开，核心观点是：

- 不把 LLM 只当检索工具，而是让它持续维护一个 Markdown Wiki
- 用 `Raw Sources / Wiki / Schema` 三层结构组织知识库
- 关键操作是 `Ingest / Query / Lint`
- Obsidian 更像 IDE，Agent 像程序员，Wiki 像代码库
- 高质量回答也应该回写成页面，形成知识复利
