# 当前Lint报告 2026-04-15

## 结论

目录结构已经基本收口，但页面写法仍然不统一。

当前优先级最高的问题不是再改目录，而是统一 frontmatter、类型枚举和页面命名。

## P1 问题

### 1. Frontmatter 键名混用

- 仍有大量页面使用 `标题`，而不是 `title`
- 这会让后续 Dataview、模板和自动化规则难以统一

疑似页面：

- `Wiki百科/1. 实体/1.3 产品/Hermes Agent.md`
- `Wiki百科/1. 实体/1.5 项目/prompts.chat.md`
- `Wiki百科/1. 实体/1.4 工具/hv-analysis.md`
- `Wiki百科/1. 实体/1.4 工具/khazix-writer.md`
- `Agent产出/3. Skills/安装日志.md`
- `个人存货/2. 学习笔记/2026-04-12 工作日报.md`

### 2. `type` 枚举不统一

发现值混用：

- `entity`
- `topic`
- `source`
- `research-question`
- `theme`
- `tool`
- `meta`

建议统一为：

- `index`
- `entity`
- `topic`
- `source`
- `rule`
- `workflow`
- `schema`
- `checklist`
- `ledger`
- `report`
- `template`
- `standard`
- `schedule`

### 3. 正式页里有脚手架或样例文本

疑似页面：

- `个人存货/1. 项目/1.4 Agent 配置.md`
- `个人存货/2. 学习笔记/2.2 参考文档/指南.md`
- `个人存货/2. 学习笔记/2.1 账户配置/指南.md`

这些页面含有示例字段或模板型占位内容，需要判定是正式文档、模板，还是应迁移到运维层。

## P2 问题

### 4. 页面命名仍有少量不稳定对象

疑似页面：

- `个人存货/3. 技能树/3.1 订阅管理/FG2gEdgZwpMkA3ApQMksnw.md`

这个文件名不具可读性，不适合作为正式长期页面名。

### 5. 分类语义仍有待复核

- `Wiki百科/2. 主题/2.4 领域知识/Gemini.md`
  当前 `type: theme`，且分类像模型对象，可能更适合作为实体或规范 topic 页。

- `Wiki百科/1. 实体/1.3 产品/Hermes Agent.md`
  结构上已归位，但语义上仍可能与 `个人存货/1. 项目/1.2 Hermes Agent.md` 有重叠，需要后续拆分“外部对象页”和“我的项目页”。

## 建议处理顺序

1. 统一 `title / type / category` 写法
2. 清理带模板痕迹的正式页
3. 修复随机文件名和命名漂移
4. 处理实体页与个人项目页的重叠对象
