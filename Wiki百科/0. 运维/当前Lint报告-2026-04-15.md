# 当前Lint报告 2026-04-15

## 结论

目录结构已经重新收口，误同步回来的平行目录和错位页面已清理。

当前优先级最高的问题是继续统一 frontmatter、类型枚举和命名。

## P1 问题

### 1. Frontmatter 键名混用

- 仍有一批页面使用 `标题`，而不是 `title`
- 这会让后续 Dataview、模板和自动化规则难以统一

疑似页面：

- `Wiki百科/1. 实体/1.4 工具/baoyu-skills.md`
- `Wiki百科/2. 主题/2.4 领域知识/Baoyu Skills.md`
- `Agent产出/1. 研究资料/hermes_agent_research/hermes_agent_research.md`

### 2. `type` 枚举不统一

发现仍有以下旧值混用：

- `entity`
- `topic`
- `source`
- `research-question`
- `meta`

本轮已修复：

- `theme -> topic`
- `tool -> entity`

仍需处理：

- `meta`

建议统一为：

- `index`
- `entity`
- `topic`
- `source`
- `rule`
- `workflow`
- `schema`
- `checklist`
- `ledger`
- `report`
- `template`
- `standard`
- `schedule`

### 3. 正式页里有脚手架或样例文本

疑似页面：

- `个人存货/1. 项目/1.4 Agent 配置.md`
- `个人存货/2. 学习笔记/2.2 参考文档/指南.md`
- `个人存货/2. 学习笔记/2.1 账户配置/指南.md`

这些页面含有示例字段或模板型占位内容，需要判定是正式文档、模板，还是应迁移到运维层。

## P2 问题

### 4. 页面命名仍有少量不稳定对象

这个文件名不具可读性，不适合作为正式长期页面名。
