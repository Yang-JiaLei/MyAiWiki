# 编辑规范

## Frontmatter 规范

统一优先使用以下字段：

```yaml
---
title:
created:
updated:
type:
category:
tags: []
aliases: []
source_status:
confidence:
---
```

## 字段规则

- `title` 统一使用 `title`，不要新写 `标题`
- `type` 只用受控值，不要混用 `theme / tool / meta`
- `updated` 每次实质修改后更新
- `tags` 只保留有检索价值的标签

## 正文结构

### 实体页

- 是什么
- 关键事实
- 相关主题
- 相关来源
- 待验证点

### 主题页

- 定义
- 核心观点
- 关键方法或机制
- 相关实体
- 相关来源
- 开放问题

### 来源页

- 来源信息
- 核心摘要
- 关键主张
- 影响到的页面
- 原文链接
- 处理状态

## 链接规范

- 首次出现的重要对象应加内部链接
- 页面末尾至少应有一组“相关页面”或“相关来源”
- 活文档不允许引用已废弃路径

## 写作规范

- 先写结论，再写展开
- 事实和推断分开
- 不直接复制大段原文作为正文
- 不用空话填充版面
- 不把临时过程写成长期结论

## 禁止项

- 同一页面混用 `title` 和 `标题`
- 同一类型页面混用多套 section 结构
- 直接把聊天内容原封不动塞进正式 Wiki 页
- 在正式页里保留明显的脚手架占位文本
