## 原则

- 原始来源只读，放在 `3. 原始来源/`
- Wiki 页面可持续改写，放在 `1. 实体/` 和 `2. 主题/`
- 每次摄入新材料，优先更新已有页面，而不是新增重复页面
- 每次高质量问答，如果有长期价值，应沉淀为页面而不是只留在聊天里
- 每次维护都要留下最小变更记录
- 活文档必须优先于历史记录修复
- 页面写法必须逐步向统一 schema 收敛

## 维护节奏

- 新资料进入时：执行一次摄入
- 主题扩展后：补交叉链接
- 每周或每批资料后：执行一次 lint
