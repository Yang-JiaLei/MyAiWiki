# 知识库运行规范

## 目标

让知识库持续增长而不失控。

这里追求的不是“多写页面”，而是：

- 分类稳定
- 入口清晰
- 来源可追溯
- 页面可维护
- 知识能复用

## 三大内容区职责

### 1. `个人存货`

- 放你自己的原生内容
- 放项目、学习笔记、技能树、思考碎片、Prompt 资产
- 不承载外部知识的正式百科页

### 2. `Agent产出`

- 放 AI 过程产物
- 放研究中间稿、对话记录、技能接入过程、系统日志、Wiki 草稿
- 不作为最终知识落点

### 3. `Wiki百科`

- 放外部知识的长期沉淀
- 分为 `实体 / 主题 / 原始来源`
- 每一条结论都应尽量可追溯到来源

## 运行原则

### 先分层，再落页

- 原始资料先登记来源
- 过程材料先归档到 `Agent产出`
- 只有稳定内容才进入 `个人存货` 或 `Wiki百科`

### 优先更新，不优先新建

- 新资料进入时，先查是否已有对应页面
- 能补现有页，就不新建平行页
- 新建页面必须能回答“为什么旧页装不下”

### 页面必须有角色

- 每个页面只能有一个主职责
- 一个页面不要同时承担来源登记、实体说明、个人记录三种角色

### 结论必须分级

- 事实、推断、假设要分开写
- 未核实内容要明确标注
- 高风险内容不得伪装成确定结论

### 入口优先于深页

- 每个区都必须有可用 README
- README 要能解释：这里放什么、不放什么、怎么查

## 变更规则

- 结构调整优先改入口和规则页
- 大规模迁移后必须补 lint
- 历史日志可以保留旧路径，不强求为历史记录追改
- 活文档中的断链和旧路径必须修复

## 准入顺序

1. 判断内容属于哪一层
2. 判断是否已有承载页面
3. 判断是正式页还是过程页
4. 补 metadata、链接、状态
5. 更新入口或台账

## 退出机制

以下内容应考虑迁移、归档或删除：

- 明显重复页
- 无来源且不可验证的外部知识页
- 已失效的过程稿
- 既不再维护也无复用价值的页面
