# 维护节奏

## 每次新增资料时

1. 判断资料进入哪一层
2. 登记来源或归档过程稿
3. 更新现有页或创建必要新页
4. 补至少一条交叉链接

## 每周维护

1. 清理断链和旧路径
2. 查重复页和命名漂移
3. 查未并入正式知识的高价值过程稿
4. 更新来源台账状态

## 每月维护

1. 检查入口页是否仍可用
2. 检查高频主题是否缺少来源支撑
3. 检查过时页面
4. 回收低价值草稿或迁移成熟内容

## 触发式维护

以下情况应立即做专项整理：

- 目录结构发生调整
- 一次性导入大量资料
- 出现重复对象页
- 外部事实发生显著变化

## 维护产出

每次维护至少产生以下之一：

- 修复后的入口页
- 新的来源登记
- 被补齐链接的正式页
- 一份 lint 结果或迁移清单
