# Wiki百科

外部知识的内化集合，基于 Karpathy LLM Wiki 模式构建。

## 结构

### 1. 实体 (Wiki百科/1. 实体/)

结构化的事实性数据。

| 子类 | 说明 | 示例 |
|------|------|------|
| 1.1 人物 | 研究者、开发者、影响者 | Andrej Karpathy, Sam Altman |
| 1.2 公司 | 组织、机构 | OpenAI, Tesla, Stanford |
| 1.3 产品 | 软件、硬件、服务 | ChatGPT, Gemini, Autopilot |
| 1.4 工具 | 软件工具、平台 | Obsidian, gemini-cli, baoyu-skills |
| 1.5 项目 | 开源项目、研究项目 | FastNodeSync, LLM-Wiki |
| 1.6 其他实体 | 未分类的实体 | 账户、服务、订阅 |

### 2. 主题 (Wiki百科/2. 主题/)

概念性、方法论和领域知识。

| 子类 | 说明 | 示例 |
|------|------|------|
| 2.1 AI | 人工智能相关概念 | LLM, Transformer, RAG |
| 2.2 技术 | 具体技术栈和架构 | Prompt Engineering, 向量数据库 |
| 2.3 方法论 | 方法和流程 | 思维框架, 最佳实践 |
| 2.4 领域知识 | 特定领域知识 | (预留) |

### 3. 原始来源 (Wiki百科/3. 原始来源/)

源文档备份，供后续处理和引用。

| 子类 | 说明 | 示例 |
|------|------|------|
| 3.1 文章 | 网络文章、博客 | prompts.chat 提示集 |
| 3.2 书籍 | 电子书、技术文档 | baoyu-skills 文档 |
| 3.3 视频 | 视频课程、演讲 | (预留) |
| 3.4 课程 | 在线课程、教程 | (预留) |

## 设计原则

- 实体：单一事物，有明确边界（一个人、一个产品）
- 主题：抽象概念，可跨多个实体
- 原始来源：未经加工的源文档，是 Wiki 的原材料
