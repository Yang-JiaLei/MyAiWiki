## 2026-04-12 — gemini-cli 认证机制分析

### 背景
安装 gemini-cli 后，运行命令未触发 OAuth 登录流程，需要分析认证机制以提供完整的使用指南。

### 调查方法
1. 从 GitHub 下载源代码包（README、配置文档、settings.schema.json）
2. 分析本地安装的 bundle 文件（/usr/local/lib/node_modules/@google/gemini-cli/bundle/）
3. 在 minified JavaScript 中搜索 OAuth 相关关键字
4. 提取 OAuth 客户端 ID、scope、回调地址

### 关键发现
- **OAuth 流程**: 使用 Google OAuth 2.0，启动本地 HTTP 服务器监听回调
- **回调地址**: http://127.0.0.1:{{随机端口}}/oauth2callback
- **Client ID**: 681255809395-oo8ft2oprdrnp9e3aqf6av3hmdib135j.apps.googleusercontent.com
- **Scope**: https://www.googleapis.com/auth/cloud-platform
- **环境变量**:
  - NO_BROWSER=true — 禁用自动打开浏览器，URL 打印到输出
  - OAUTH_CALLBACK_HOST — 服务器监听地址（默认 127.0.0.1）
  - OAUTH_CALLBACK_PORT — 回调端口（默认随机）
- **限制**: OAuth 需要图形界面浏览器，在无头服务器环境无法完成

### 结论
在服务器/容器环境，**必须使用 API Key 方式**（GEMINI_API_KEY 环境变量）。OAuth 仅适用于本地桌面使用。
